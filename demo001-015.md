# Java 后端循序渐进教学项目（demo001 - demo015）

## 系列目标

通过 15 个逐步演进的 Demo，从最原始的 JDBC 写法，一步步过渡到现代 Spring Boot + MyBatis-Plus + REST API 的生产级开发方式。每一步只做一个重点变化，方便学习和对比。

## 详细版本说明

### demo001 - 最原始的 JDBC
所有 CRUD 逻辑写在一个类中，使用原生 JDBC + Scanner 实现控制台菜单。  
**目的**：让你彻底看清楚 Connection、PreparedStatement、ResultSet 到底在做什么。

### demo002 - 传统分层架构
引入 Maven，拆分为 Entity、Mapper、Service、Controller 层，所有 getter/setter 手动编写。  
**目的**：理解分层思想和“手动装配”的麻烦。

### demo003 - 引入 Lombok（最小改动）
仅在 Entity 上使用 Lombok（@Data + @Builder 等）。  
**目的**：展示“只改一个文件”就能大幅减少样板代码。

### demo004 - 加入日志（@Slf4j）
在 demo003 基础上增加日志能力。  
**目的**：体现在已有 Lombok 的项目上，再增加新功能的渐进方式。

### demo005 - MyBatis XML 替换 Mapper
用 MyBatis XML 完全替代手写的 JDBC MapperImpl。  
**目的**：第一次接触 ORM，SQL 写在 XML 中。

### demo006 - MyBatis 注解方式
把 XML 中的 SQL 迁移到接口上的 `@Select`、`@Insert` 等注解。  
**目的**：对比 MyBatis 的两种主流写法。

### demo007 - MyBatis-Spring 集成
引入 Spring + MyBatis-Spring，彻底告别手动 `SqlSession` 的写法。  
**目的**：进入依赖注入时代。

### demo008 - Spring Boot 最终版
使用 Spring Boot 自动配置，抛弃 `applicationContext.xml`，改用 `application.yml`。  
**目的**：掌握现代 Spring Boot 项目标准结构。

### demo009 - MyBatis-Plus BaseMapper
Mapper 直接继承 `BaseMapper<Student>`，不再手写任何 CRUD 方法。  
**目的**：感受 MyBatis-Plus 对 Mapper 层的极致简化。

### demo010 - MyBatis-Plus Service 封装
Service 层也继承 `ServiceImpl`，进一步减少业务代码。  
**目的**：MyBatis-Plus 在 Service 层的威力。

### demo011 - 统一返回结果 + 全局异常处理
引入 `Result<T>` 统一响应格式 + `GlobalExceptionHandler` 集中处理异常。  
**目的**：建立生产级项目的响应和异常规范。

### demo012 - RESTful API 改造
彻底移除控制台菜单逻辑，改用 `@RestController` 提供 HTTP 接口。  
**目的**：从控制台程序转型为真正的 Web API 项目。

### demo013 - 参数校验（Validation）
引入 DTO + `@Valid` + 校验注解，增强 `GlobalExceptionHandler` 处理校验错误。  
**目的**：让接口参数校验自动化、规范化。

### demo014 - 分页 + 条件查询
使用 MyBatis-Plus 的 `Page` 和 `LambdaQueryWrapper` 实现分页和动态搜索。  
**目的**：掌握真实项目中最常用的列表查询方式。

### demo015 - 事务管理（@Transactional）
新增批量新增接口，使用 `@Transactional` 演示事务回滚机制。  
**目的**：理解如何保证数据一致性。
